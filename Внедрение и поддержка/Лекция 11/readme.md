## 🖥️ Часть 1. Организация одноранговой локальной сети (Peer-to-Peer)

**Задача:** Физически и логически связать две рабочие станции для беспрепятственного обмена файлами.

**1. Физический уровень:** Соединение реализовано напрямую (без использования коммутатора) посредством кабеля «витая пара» через интерфейсы Ethernet до характерного щелчка фиксатора.

**2. Сетевой уровень (IP-конфигурация):** Настройка выполнялась через оснастку «Центр управления сетями и общим доступом».

| Параметр | Рабочая станция №1 | Рабочая станция №2 |
| :--- | :--- | :--- |
| **IP-адрес (IPv4)** | `192.168.0.1` | `192.168.0.2` |
| **Маска подсети** | `255.255.255.0` | `255.255.255.0` |
| **Основной шлюз** | *Не требуется* | `192.168.0.1` |
| **Имя узла (Hostname)** | `Comp1` | `Comp2` |
| **Рабочая группа** | `WORKGROUP` | `WORKGROUP` |

**3. Настройка прав доступа и файлового обмена:**
* В профиле сети активировано «Сетевое обнаружение» и функция общего доступа к файлам/принтерам.
* Отключен запрос пароля при попытке входа с соседнего компьютера.
* Для тестовой директории настроен расширенный доступ (Вкладка «Доступ» -> «Отдельные люди»). Права на чтение и запись выданы группе пользователей «Все».

**4. Верификация:** Работоспособность проверена отправкой ICMP-пакетов командой `ping 192.168.0.1`. Доступ к файловому ресурсу успешно осуществлен через файловый менеджер по UNC-пути: `\\Comp1`.

## ⚙️ Часть 2. Администрирование сети через утилиты командной строки

В ходе выполнения лабораторной работы были изучены и применены на практике консольные сетевые инструменты Windows.

* **Вызов мануала:** Для любой команды справка вызывается ключом `/?` (например, `getmac /?`).

**Анализ аппаратных адресов (`getmac`)**
Применение команды с ключом детализации `/v` позволило выгрузить список всех физических (MAC) адресов устройства в формате `XX-XX-XX-XX-XX-XX` с привязкой к конкретным сетевым адаптерам.

**Управление конфигурацией (`netsh`)**
Утилита Network Shell использовалась для глубокой настройки интерфейсов:
* Команда `netsh dump` вывела текущую конфигурацию сети.
* Произведено назначение статического IP-адреса через консоль: 
  `netsh interface ipv4 add address name="Подключение по локальной сети" address=192.168.137.5 mask=255.255.255.0 gateway=192.168.137.1`
* Выполнена смена адреса "на лету" параметром `set address`.

**Управление сетевыми ресурсами (`net`)**
* `net use` — выведен список примонтированных сетевых томов (дисков).
* `net user` — проведен аудит локальных учетных записей операционной системы.
* `net share` — получен перечень всех открытых общих ресурсов, включая скрытые системные директории (с символом `$`).

**Расширенная трассировка (`pathping`)**
Выполнена маршрутизация до внешнего IP-адреса `172.21.66.57`. В отличие от обычного ping, утилита собрала детальную статистику по каждому промежуточному узлу (хопу), показав процент потерянных пакетов на магистрали.

**Настройка безопасности (`netsh advfirewall`)**
* Отработано глобальное отключение брандмауэра: `set allprofiles state off` и включение конкретных профилей (domain/private).
* Задана строгая политика безопасности: `set allprofiles firewallpolicy blockinbound,allowoutbound` (запрет входящих, разрешение исходящих).
* Добавлено исключение (правило) для программы: `firewall add rule name="radmin" dir=in action=allow program="C:\Path\radmin.exe"`.

## ❓ Теоретический блок (Контрольные вопросы)

**1. Что такое IP-адрес и какие виды существуют?**
IP-адрес — это уникальный логический номер устройства в сети. Выделяют две основные версии: **IPv4** (32 бита, состоит из 4 чисел) и **IPv6** (128 бит, использует шестнадцатеричную систему). По способу получения они делятся на статические (прописываются администратором вручную) и динамические (выдаются сервером DHCP автоматически).

**2. В чем главное отличие IPv6 от IPv4?**
Стандарт IPv6 был разработан из-за глобальной нехватки адресов формата IPv4. Благодаря 128-битной архитектуре он обеспечивает практически неисчерпаемое количество уникальных адресов, улучшенную маршрутизацию и встроенную поддержку шифрования (IPsec).

**3. Для чего нужны утилиты `netsh` и `getmac`?**
* `netsh` — это мощнейшая консольная оболочка для конфигурирования сети. Она позволяет менять IP-адреса, управлять брандмауэром и сбрасывать кэш интерфейсов без использования графического меню Windows.
* `getmac` — узкоспециализированная утилита для получения физического адреса (MAC-адреса) сетевой карты, «зашитого» в нее на заводе.

**4. В чем разница результатов `ping` и `pathping`?**
Команда `ping` просто констатирует факт: есть связь с узлом или нет (плюс показывает скорость ответа). `pathping` — это инструмент глубокой аналитики. Он показывает весь маршрут прохождения сигнала и вычисляет, на каком именно промежуточном сервере происходят задержки или потери пакетов.

**5. Что такое Брандмауэр Windows?**
Это программный сетевой экран (фаервол), который фильтрует весь входящий и исходящий трафик. Он защищает компьютер от сетевых атак, несанкционированных подключений и троянов, блокируя соединения, которые не соответствуют установленным правилам безопасности.

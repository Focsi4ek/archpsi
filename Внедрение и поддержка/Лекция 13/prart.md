# Отчет по лабораторной работе №15
**Тема:** Настройка и управление Центром обновления в ОС Windows 10
**Цель работы:** Практическое изучение механизмов обновления операционной системы Windows 11 и их настройка.

## Практическая часть

### 1. Доступ к Центру обновления
Для начала работы я зашел в системные настройки: **Пуск → Параметры → Обновление и безопасность**. Далее выбрал раздел **«Центр обновления Windows»**.
После нажатия на кнопку проверки обновлений, система инициировала поиск актуальных пакетов.

### 2. Анализ доступных обновлений
По результатам поиска система предложила два типа пакетов:
* **Критические (важные)** — патчи безопасности и исправления системных ошибок.
* **Дополнительные (необязательные)** — свежие версии драйверов и вспомогательные утилиты.
Информацию об обязательных пакетах я проверил, перейдя по пути: Параметры → Обновление и безопасность → **Просмотр журнала обновлений**.

### 3. Процесс установки
Я дождался окончания загрузки и инсталляции найденных пакетов. В журнале обновлений статус каждого загруженного патча безопасности изменился на **«Успешно установлено»**, что подтверждает корректность операции.

### 4. Блокировка обновлений через GPO (Групповые политики)
1. Запустил редактор локальной групповой политики комбинацией `Win+R` и командой `gpedit.msc`.
2. Перешел по ветке: **Конфигурация компьютера → Административные шаблоны → Компоненты Windows → Центр обновления Windows**.
3. Открыл свойства политики **«Настройка автоматического обновления»** и перевел ее в состояние **«Отключено»**.

В результате в интерфейсе Центра обновления появилась предупреждающая надпись: *«Некоторыми параметрами управляет ваша организация»*, подтверждающая успешное отключение фонового поиска обновлений.

### 5. Локальная установка скачанных обновлений
Через локальную сеть я подключился к директории `\\kbastrikin`. Оттуда были скопированы установочные файлы в форматах `.msu` и `.cab`. Их инсталляция производилась вручную.
Характеристики каждого файла были уточнены через Каталог Центра обновления Майкрософт.

### 6. Конфигурация службы Windows Update
Перешел в управление службами: **Панель управления → Администрирование → Управление компьютером → Службы и приложения → Службы**.
В списке нашел службу `wuauserv` (Центр обновления Windows) и изменил ее свойства:
* В разделе **«Общие»** задал тип запуска: **«Вручную»**.
* В разделе **«Восстановление»** настроил действия при сбоях: первый и второй отказ — «Перезапуск службы» (интервал 1 минута), последующие отказы — «Перезагрузка компьютера».

Сохранил изменения кнопками **Применить** и **ОК**.

## Ответы на контрольные вопросы

### 1. Понятие обновления ОС
**Обновление** представляет собой программный патч от разработчика, предназначенный для устранения багов, закрытия уязвимостей, а также добавления новых функций. В ОС семейства Windows за их доставку и установку отвечает специализированная служба Windows Update.

### 2. Классификация обновлений
* **Важные / Критические:** Закрывают уязвимости и чинят критические системные ошибки. Устанавливаются в автоматическом режиме.
* **Рекомендуемые:** Улучшают стабильность и производительность, но не являются критически важными. Могут ставиться автоматически или вручную.
* **Необязательные:** Драйверы периферии, дополнительные языки. Устанавливаются строго по инициативе пользователя.
* **Накопительные обновления (Cumulative Updates):** Современная замена старым Service Pack в Windows 10. Содержат все ранее вышедшие исправления единым накопительным пакетом.

### 3. Назначение Центра обновления Windows
**Центр обновления Windows (Windows Update)** — это базовый компонент системы, обеспечивающий актуализацию программного обеспечения.
Его основные функции:
* Поиск, загрузка и инсталляция новых патчей.
* Ведение истории установленных пакетов.
* Настройка расписания для установки и перезагрузки.
* Выбор необязательных компонентов (например, сторонних драйверов).

На уровне системы этот инструмент представлен службой `wuauserv`. Для глубокой настройки или полного отключения компонента применяются локальные политики в оснастке `gpedit.msc`.

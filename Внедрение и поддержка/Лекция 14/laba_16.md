# Отчет по практической работе №16
**Тема:** Управление групповыми политиками в ОС Windows 10
**Цель работы:** Ознакомиться с основами функционирования групповых политик в Windows, а также получить практические навыки работы в редакторе локальной групповой политики.

---

## Теоретические сведения

### Запуск инструмента редактирования
Вызвать редактор локальной групповой политики можно через диалоговое окно «Выполнить» (комбинация `Win + R`) вводом команды `gpedit.msc` и нажатием клавиши Enter. Также утилиту можно найти через системный поиск в меню «Пуск».

### Структура интерфейса
Рабочее окно программы разделено на две основные области:

| Область интерфейса | Описание содержимого |
| :--- | :--- |
| **Левая панель** | Дерево каталогов с иерархией политик |
| **Правая панель** | Перечень конкретных параметров в выбранной папке |

Глобально все настройки делятся на два узла:
*   **Конфигурация компьютера** — параметры применяются ко всему ПК в целом, активируются еще на этапе загрузки ОС до авторизации.
*   **Конфигурация пользователя** — настройки привязаны к конкретному профилю и загружаются только после входа учетной записи в систему.

Каждый из узлов имеет три стандартных подраздела:

| Подраздел | Что настраивается |
| :--- | :--- |
| **Конфигурация программ** | Опции установленного ПО |
| **Конфигурация Windows** | Системные настройки и политики безопасности |
| **Административные шаблоны** | Удобный графический интерфейс для управления ключами системного реестра |

### Состояния параметров
Для каждой политики доступен один из трех статусов:
*   **Не задано** — настройка находится в состоянии по умолчанию (не применяется).
*   **Включено** — параметр активирован.
*   **Отключено** — параметр принудительно деактивирован.

### Инструменты поиска
Поскольку политик около трех тысяч, для удобства предусмотрен фильтр:
1. Переходим в **Административные шаблоны → Все параметры**.
2. В верхнем меню выбираем **Действия → Параметры фильтра**.
3. Ставим галочку **«Включить фильтры по ключевым словам»**.
4. Вписываем искомое слово и задаем условие поиска (Любое слово / Все слова / Точная фраза).
5. Жмем **OK** и активируем сам **Фильтр** на панели.

## Ход выполнения практических заданий

### Задание 1. Редактирование экрана безопасности (Ctrl+Alt+Delete)
**Расположение:** Конфигурация пользователя → Административные шаблоны → Система → Варианты действий после нажатия CTRL+ALT+DEL.

Какие элементы можно скрыть:

| Название политики | Что пропадает с экрана |
| :--- | :--- |
| Запретить изменение пароля | Кнопка «Сменить пароль» |
| Запретить блокировку компьютера | Кнопка «Блокировать компьютер» |
| Удалить диспетчер задач | Кнопка «Запустить диспетчер задач» |
| Запретить завершение сеанса | Кнопка «Выйти из системы» |

**Действия:**
1. Запустил утилиту `gpedit.msc`.
2. Прошел по указанному пути в дереве каталогов.
3. Выбрал нужные политики, двойным кликом открыл свойства и перевел их в статус **«Включено»**, нажал ОК.
4. Нажал Ctrl+Alt+Delete — проверил, что выбранные пункты исчезли из меню.

**Альтернатива через реестр** (`HKCU\Software\Microsoft\Windows\CurrentVersion\Policies\System`):
`DisableChangePassword = 0`
`DisableLockWorkstation = 0`
`DisableTaskMgr = 0`
Скрытие кнопки выхода настраивается в ветке `HKCU\...\Policies\Explorer` ключом `NoLogoff = 1`.

---

### Задание 2. Изменение панели мест (окна Открыть/Сохранить)
**Расположение:** Конфигурация пользователя → Административные шаблоны → Компоненты Windows → Проводник Windows → Общее диалоговое окно открытия файлов.

**Действия:**
1. Нашел параметр **«Элементы, отображаемые в панели мест»**.
2. Изменил статус на **«Включено»**.
3. В строках «Элемент 1» – «Элемент 5» прописал пути к необходимым локальным и сетевым директориям.
4. Сохранил настройки.

**Альтернатива через реестр** (`HKCU\Software\Microsoft\Windows\CurrentVersion\Policies\comdlg32\Placesbar`):
Ключи `Place0` ... `Place4` со значениями нужных путей.

---

### Задание 3. Настройка списка «Недавние документы»
**Расположение:** Конфигурация пользователя → Административные шаблоны → Компоненты Windows → Проводник Windows.

**Действия:**
1. Открыл свойства политики **«Максимальная длина списка "Недавние документы"»**.
2. Активировал параметр (**«Включено»**).
3. Указал необходимое количество отображаемых файлов (доступен диапазон 1-9999).
4. Нажал ОК. 
*(Примечание: Базовое значение в Windows — 15 документов. При превышении лимита отображения появляется скролл-бар).*

---

### Задание 4. Активация трекера завершения работы
**Расположение:** Конфигурация компьютера → Административные шаблоны → Система.

**Действия:**
1. Открыл параметр **«Отображать диалог слежения за завершением работы»**.
2. Перевел в состояние **«Включено»**.
3. В выпадающем меню выбрал параметр **«Всегда»**.
4. Сохранил изменения.

Теперь при попытке выключить ПК появляется окно с обязательными полями:
*   **Запланированное** (чекбокс планирования)
*   **Причина** (выбор из выпадающего списка)
*   **Комментарий** (поле для ручного ввода текста)

**Альтернатива через реестр** (`HKLM\Software\Policies\Microsoft\Windows NT\Reliability`):
Ключи `ShutdownReasonOn = 1` и `ShutdownReasonUI = 1`.

## Ответы на контрольные вопросы

### 1. Понятие групповых политик
**Групповые политики (GPO)** — это встроенный набор правил и параметров для централизованного управления настройками ОС, правами пользователей и рабочим пространством. Оснастка `gpedit.msc` доступна только в старших редакциях ОС (Pro, Enterprise, Education/Ultimate) и является более безопасной альтернативой прямому редактированию реестра.

### 2. Главные категории политик
| Категория | Особенность применения |
| :--- | :--- |
| **Конфигурация компьютера** | Задает глобальные правила для конкретного ПК. Работает для всех профилей, применяется на этапе загрузки ОС. |
| **Конфигурация пользователя** | Задает правила для конкретного пользователя. Применяется только в момент его авторизации в системе. |

### 3. Структура разделов
Каждая глобальная категория разбита на три логических раздела:
*   **Конфигурация программ:** управление установленными приложениями.
*   **Конфигурация Windows:** базовые параметры системы, скрипты, настройки безопасности.
*   **Административные шаблоны:** графическая оболочка для изменения тысяч ключей реестра.

### 4. Запуск редактора
Инструмент запускается вводом `gpedit.msc` в окне команд (Win+R) или через меню Пуск.
Термин «локальная» в названии указывает на то, что вносимые изменения будут действовать только на текущем физическом компьютере (в отличие от доменных политик Active Directory).

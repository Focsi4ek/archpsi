# Теоретическая справка: Групповые политики в Windows

## Что такое групповая политика (GPO)

**Групповая политика** представляет собой комплекс настроек, предназначенных для управления рабочей средой как отдельного пользователя, так и операционной системы в целом. 

Это важнейший инструмент системного администрирования в среде Windows, который позволяет гибко настраивать систему — начиная от визуального оформления интерфейса и заканчивая строгими ограничениями прав доступа к системным функциям.

> ⚠️ **Важное замечание:** Данный функционал не ориентирован на домашнее использование. Оснастка для локального редактирования политик присутствует исключительно в старших редакциях ОС: **Professional (Pro)**, **Enterprise** и **Ultimate**.

---

## Сферы применения политик

Групповые политики применяются к двум базовым сущностям:

| Сущность (объект) | Особенности применения |
| :--- | :--- |
| **Пользователи** | Задают параметры рабочей среды для конкретной учетной записи. Эти настройки действуют для юзера независимо от того, за какой именно компьютер в сети он сел. |
| **Компьютеры** | Жестко фиксируют системные настройки для конкретной физической машины. Они будут действовать для абсолютно любого пользователя, авторизовавшегося на этом ПК. |

---

## Элементы пользовательского окружения

Что именно можно контролировать с помощью политик:
*   Внешний вид и параметры рабочего стола.
*   Перечень разрешенных или запрещенных для запуска программ.
*   Автоматический запуск скриптов при входе в учетную запись или при ее завершении.
*   Уровни прав и доступы к локальным ресурсам (файлам, папкам, принтерам).

В корпоративной среде (например, в доменах Active Directory) для централизованного управления всеми этими правилами используется именно **механизм групповых политик**.

---

## Понятие объекта GPO

**Объект групповой политики (Group Policy Object, GPO)** — это базовая логическая единица управления, которая хранится в системе как самостоятельный элемент.

*   Каждому такому объекту присваивается свой глобальный уникальный идентификатор — **GUID**.
*   Поскольку архитектура Windows изначально является многопользовательской, GPO позволяет формировать изолированную и уникальную среду для каждой отдельной учетной записи.

## Способ вызова редактора

Быстрый запуск оснастки осуществляется через окно «Выполнить»:
```
Win + R → ввести команду gpedit.msc → нажать Enter
```

---

## Архитектура локального редактора

Глобально все доступные параметры в окне программы разбиты на два больших узла:

### 1. Конфигурация компьютера
Охватывает глобальные параметры физической машины. Эти правила применяются к системе в целом и работают **совершенно одинаково для всех**, вне зависимости от того, кто именно вошел в систему.

### 2. Конфигурация пользователя
Применяется персонально к **конкретной учетной записи** и содержит параметры, которые загружаются только при работе данного юзера.

## Внутренние категории (присутствуют в обеих ветвях)

Каждый из двух главных разделов содержит стандартизированные папки:

| Категория | Описание содержимого |
| :--- | :--- |
| **Конфигурация программ** | Опции для управления инсталлированным программным обеспечением. |
| **Конфигурация Windows** | Системные параметры, скрипты, а также детальные настройки политик безопасности. |
| **Административные шаблоны** | Удобный графический интерфейс для управления ключами системного реестра. По сути, это те же параметры, что и в `regedit`, но структурированные и переведенные на понятный язык. |

# Отчет по лабораторной работе №17
**Тема:** Миграция пользовательских данных между ОС Windows 7 и Windows 10
**Цель работы:** Практическое освоение алгоритма переноса файлов со старой операционной системы (Windows 7) на современную (Windows 10) с применением утилиты «Средство переноса данных».

---

## Используемое оборудование
*   **Исходная машина:** ПК под управлением Windows 7 Professional.
*   **Целевая машина:** Виртуальная среда (VM) с установленной ОС Windows 10 Pro.

---

## Практическая часть

### Этап 1. Подготовка файлов в среде Windows 7
1. В корне системы была создана директория с именем **«For Transferring»**.
2. Внутри нее с помощью стандартного Блокнота создан текстовый документ **Data** с проверочной фразой *«Со старого ПК»*.
3. На рабочем столе заранее подготовлена папка **«Перенос»**, предназначенная для сохранения итогового архива миграции.

### Этап 2. Экспорт данных с помощью штатной утилиты
**Расположение инструмента:** Пуск → Все программы → Стандартные → Служебные → **Средство переноса данных Windows**.

**Алгоритм действий:**
1. На стартовом экране мастера нажал **«Далее»**.
2. В качестве типа носителя выбрал **«Внешний диск или USB-устройство флэш-памяти»**.
3. Подтвердил статус ПК, выбрав пункт **«Это мой исходный компьютер»**.
4. Дождался окончания сканирования. В окне выбора данных убрал галочки со всех профилей и перешел в **«Настройку»** своей учетной записи.
5. Через кнопку **«Дополнительно»** вручную указал путь к созданной ранее папке `For Transferring`, отметил только файл **Data** и нажал **«Сохранить»**.
6. Шаг с установкой пароля пропустил (оставил поля пустыми).
7. Указал путь для сохранения бэкапа — папка **«Перенос»** на рабочем столе. 

**Результат:** Утилита сформировала файл миграции с расширением `.MIG` (размер составил около 1 КБ).

### Этап 3. Подготовка целевой системы (Windows 10)
1. Осуществлен запуск виртуальной машины с Windows 10.
2. Для корректного обмена файлами между хостом и гостевой ОС были установлены дополнения виртуальной машины (VMware Tools / VirtualBox Guest Additions) с последующей перезагрузкой.
3. Папка **«Перенос»** с файлом `.MIG` скопирована с исходного ПК на рабочий стол Windows 10.

### Этап 4. Восстановление (импорт) данных
Поскольку Microsoft удалила утилиту переноса из Windows 10, для распаковки данных потребовался запуск старой версии программы:
1. Каталог **migwiz** (взятый из папки с отчетами) был скопирован на рабочий стол целевой ОС.
2. Произведен запуск исполняемого файла из папки `migwiz`.
3. В открывшемся мастере восстановления я указал путь к скопированному файлу `.MIG`.
4. Процесс импорта завершился успешно. Файл Data распаковался в нужную директорию.

## Ответы на контрольные вопросы

### 1. Успешно ли прошел перенос файлов?
Да, миграция прошла без ошибок. Тестовый файл **Data** был корректно запакован утилитой в Windows 7, перенесен в виде архива `.MIG` и успешно распакован в среде Windows 10 с помощью портативной папки `migwiz`. Содержимое файла не пострадало.

### 2. Какие существуют способы миграции данных?

| Метод миграции | Принцип работы |
| :--- | :--- |
| **Утилита migwiz (.MIG)** | Создание специального контейнера с файлами и настройками профиля пользователя (штатный метод старых версий Windows). |
| **Ручное копирование** | Использование внешних HDD, SSD или USB-флешек для банального переноса файлов Ctrl+C / Ctrl+V. |
| **Сетевые папки (SMB)** | Расшаривание директорий в локальной сети и передача данных напрямую между ПК. |
| **Облачные сервисы** | Синхронизация через OneDrive, Google Диск или Яндекс.Диск. Данные выгружаются в облако на старом ПК и скачиваются на новом. |
| **Кабель Easy Transfer** | Физическое соединение двух компьютеров специальным USB-кабелем для прямой передачи. |
| **Бэкап-системы** | Создание полного образа системы или резервных копий через стороннее ПО (Acronis, Macrium) с последующим выборочным восстановлением. |
| **Общие папки ВМ** | Настройка Shared Folders в гипервизоре для моментального доступа к файлам хост-системы из виртуальной машины. |

# Теоретическая справка: Технологии резервного копирования и восстановления

## Суть резервного копирования

**Резервное копирование (Backup)** — это процедура создания дубликатов цифровой информации на независимых носителях. Главная цель этого процесса — обеспечение возможности быстро вернуть данные на исходное место в случае их случайного удаления, порчи или аппаратного сбоя.

**Основные факторы утраты данных:**
*   Физический выход из строя оборудования (жестких дисков, SSD).
*   Активность вредоносного программного обеспечения (вирусы, шифровальщики).
*   Человеческий фактор (ошибочные действия пользователя или администратора).

---

## Базовые принципы бэкапа файловых систем

Современные файловые системы содержат встроенные механизмы, помогающие отслеживать изменения для корректного архивирования. Программы резервного копирования ориентируются на специальные атрибуты файлов:

*   **Метка времени** — дата и точное время последней модификации документа.
*   **Атрибут «Архивный»** — специфический маркер (особенно актуально для ОС семейства Windows), который автоматически выставляется при любом изменении файла, сигнализируя системе бэкапа, что этот объект нуждается в сохранении.

---

## Классификация методов резервного копирования

### 1. Нормальное (Полное / Normal)
*   Сохраняет абсолютно **все** файлы, выбранные для задания, независимо от их атрибутов.
*   Является отправной точкой для любого цикла бэкапов.
*   Для полного восстановления системы требуется распаковать только **один этот (последний)** архив.

### 2. Инкрементное (Добавочное / Incremental)
*   Захватывает только те данные, которые менялись **после создания любой последней копии** (неважно, полной или инкрементной).
*   Атрибут «Архивный» у скопированных файлов **очищается**.
*   Восстановление: сначала разворачивается последняя **нормальная** копия, а затем строго по очереди накатываются **все инкрементные** архивы.

### 3. Дифференциальное (Разностное / Differential)
*   Сохраняет все файлы, измененные строго **с момента создания последнего нормального (полного)** бэкапа.
*   Атрибут «Архивный» **не очищается** (сохраняется).
*   Позволяет экономить место по сравнению с полным бэкапом, но файлы дублируются каждый день до следующего нормального бэкапа.
*   Восстановление: разворачивается базовая **нормальная** копия, а затем только **один (самый свежий) разностный** архив.

### 4. Зеркальное копирование (Простое / Copy)
*   Аналог полного бэкапа — копируются **все** указанные файлы.
*   Главное отличие: атрибут «Архивный» **не сбрасывается**. Это значит, что создание зеркальной копии никак не сбивает настроенное расписание инкрементных или дифференциальных бэкапов.
*   Для восстановления требуется только этот архив.

---

## Сравнение режимов копирования

| Режим (Тип) | Объем сохраняемых данных | Сброс атрибута «Архивный» | Процесс восстановления |
| :--- | :--- | :--- | :--- |
| **Нормальный** | Абсолютно все выбранные файлы | Да | Достаточно последнего нормального архива |
| **Инкрементный** | Только изменения с момента *любого* последнего бэкапа | Да | Нормальный архив + цепочка *всех* инкрементных копий |
| **Дифференциальный** | Только изменения с момента последнего *нормального* бэкапа | Нет | Нормальный архив + *один* последний дифференциальный |
| **Зеркальный** | Абсолютно все выбранные файлы | Нет | Достаточно этого архива |

## Технология непрерывной защиты (Точки мгновенного восстановления)

*   В отличие от классических методов, копии здесь создаются **вне расписания** — процесс идет непрерывно в фоновом режиме.
*   При сохранении любого файла на диск, система автоматически и асинхронно дублирует его в другое хранилище (чаще всего по локальной сети).
*   Хотя это создает дополнительную нагрузку на дисковую подсистему, метод позволяет полностью отказаться от составления графиков бэкапов.
*   Главное преимущество: возможность откатить систему на **любую секунду** в прошлом.

## Архивирование состояния системы (System State)

ОС Windows обладает встроенной функцией создания слепка системных параметров. Это критически важный архив для восстановления работоспособности ОС после серьезных сбоев.

**Что входит в состав архива System State:**
*   Файлы системного реестра.
*   Служебные файлы загрузчика Windows.
*   База данных регистрации классов (COM+).
*   *На серверах сертификации:* База данных службы сертификатов.
*   *На контроллерах домена:* База данных каталога Active Directory и системная директория SYSVOL.

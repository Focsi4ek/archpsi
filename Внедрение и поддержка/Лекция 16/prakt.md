# Отчет по лабораторной работе №18
**Тема:** Использование безопасного режима для восстановления ОС Windows
**Цель работы:** Получить практические навыки диагностики и восстановления работоспособности операционной системы с использованием безопасного режима (Safe Mode).

---

## Теоретические сведения

**Безопасный режим (Safe Mode)** — это специальный диагностический вариант старта операционной системы. В этом состоянии загружаются только критически важные службы и базовые драйверы, необходимые для работы системы. Все сторонние программы, элементы автозагрузки и визуальные эффекты игнорируются.

**Основные сценарии применения:**

| Задача | Обоснование использования |
| :--- | :--- |
| **Антивирусная проверка** | Вредоносное ПО не может запуститься из автозагрузки, что позволяет антивирусу беспрепятственно найти и изолировать угрозу. |
| **Откат системы (Восстановление)** | В случаях, когда ОС зависает или перезагружается в обычном режиме, Safe Mode обеспечивает стабильную среду для запуска инструмента восстановления. |
| **Работа с драйверами** | Загружаются только стандартные драйверы (например, базовый видеодрайвер VGA). Это позволяет удалить сбойные драйверы стороннего оборудования без синих экранов (BSOD). |
| **Аппаратная/программная диагностика** | Если в Safe Mode система работает стабильно, проблема кроется в софте или драйверах. Если сбои продолжаются — вероятна аппаратная неисправность. |

## Практическая часть

### Этап 1. Настройка принудительной загрузки в Safe Mode
1. Находясь в штатном режиме Windows 7, я открыл окно «Выполнить» (`Win+R`) и ввел команду `msconfig`.
2. В окне «Конфигурация системы» перешел на вкладку **«Загрузка»**.
3. В блоке параметров установил маркер напротив пункта **«Безопасный режим»**.
4. Нажал **Применить** и **ОК**, после чего в появившемся диалоге выбрал **«Выход без перезагрузки»**.

### Этап 2. Генерация контрольной точки восстановления
1. Через меню Пуск открыл поиск и ввел запрос **«Создание точки»**.
2. В появившемся окне свойств системы (вкладка **«Защита системы»**) обратился к блоку параметров защиты.
3. Нажал кнопку **«Создать»**.
4. В качестве имени точки восстановления ввел свои идентификационные данные: *[Абрамов_28Ипо8481]*.
5. Нажал **«Создать»** и дождался системного уведомления об успешном завершении процесса.

### Этап 3. Контроль наличия созданной точки
1. Через поиск в меню Пуск запустил утилиту **«Восстановление системы»**.
2. В стартовом окне мастера нажал **«Далее»**.
3. В предложенном перечне контрольных точек нашел запись со своим именем.
4. Убедившись в наличии бэкапа, нажал **«Отмена»** для прерывания процедуры на данном этапе.

### Этап 4. Вход в безопасный режим
1. Инициировал стандартную перезагрузку ПК.
2. Благодаря настройкам `msconfig` (из Этапа 1), система автоматически стартовала в безопасном режиме.
3. *Визуальные признаки:* черный фон рабочего стола, пониженное разрешение экрана, информационные надписи «Безопасный режим» по углам монитора.

### Этап 5. Исследование точек восстановления в диагностической среде
1. Повторно запустил инструмент **«Восстановление системы»**.
2. Активировал опцию **«Показать другие точки восстановления»**.
*Зафиксированное количество автоматических точек:* 3.
3. Выделил созданную мной точку и нажал **«Поиск затрагиваемых программ»**.
*Результат:* Список оказался пустым, так как после создания бэкапа новые программы не устанавливались.
4. Вернулся к списку, выбрал **самую старую** из доступных точек и повторил поиск.
*Результат:* Список содержит множество записей об обновлениях и компонентах, так как с момента ее создания конфигурация системы менялась значительно сильнее.

### Этап 6. Процедура отката системы
1. В мастере выбрал свою именную точку *[Абрамов_28Ипо8481]* и перешел **«Далее»** → **«Готово»**.
2. Подтвердил запуск операции в предупреждающем окне (нажал **«Да»**).
3. Стартовал процесс инициализации восстановления, за которым последовала автоматическая перезагрузка.
4. ПК вновь загрузился в безопасном режиме и выдал уведомление: **«Восстановление системы выполнено успешно»**.

### Этап 7. Отключение диагностического запуска
1. Снова вызвал утилиту `msconfig` и открыл вкладку **«Загрузка»**.
2. Снял ранее установленную галочку **«Безопасный режим»**, подтвердил действия (**Применить** → **ОК**).
3. Перезагрузил компьютер — система корректно стартовала в штатном режиме.

## Ответы на контрольные вопросы

### 1. Суть безопасного режима
**Безопасный режим** — это отладочное состояние Windows, предназначенное для локализации и устранения программных сбоев. В этом состоянии ОС блокирует запуск сторонних драйверов, утилит из автозагрузки и второстепенных системных служб. Запускается только "голый" каркас системы.
Визуальные отличия: отсутствие обоев, базовое разрешение экрана, водяные знаки по краям дисплея. 

### 2. Способы активации
| Метод запуска | Описание |
| :--- | :--- |
| **Клавиша F8** | Классический способ (до Windows 8). Необходимо нажимать F8 сразу после POST-экрана BIOS для вызова меню дополнительных вариантов загрузки. |
| **Утилита msconfig** | Настройка через графический интерфейс (вкладка «Загрузка» → активация соответствующего чекбокса). Гарантирует запуск в Safe Mode даже при включенной функции быстрого старта. |
| **Особые варианты загрузки** | Актуально для Windows 10/11. Путь: Параметры → Обновление и безопасность → Восстановление → Перезагрузить сейчас. |

В расширенном меню загрузки доступны три вариации:
*   Стандартный безопасный режим (без интернета).
*   С поддержкой сетевых драйверов (с доступом в локальную сеть и интернет).
*   С поддержкой командной строки (вместо привычного проводника запускается консоль `cmd.exe`).

### 3. Сравнение диагностических режимов

| Характеристика | Безопасный режим (Safe Mode) | Чистая загрузка (Clean Boot) |
| :--- | :--- | :--- |
| **Назначение** | Критическое восстановление неработающей ОС, удаление вирусов и сбойных драйверов. | Поиск конфликтов между сторонними программами при работающей системе. |
| **Драйверы** | Используются только базовые, встроенные в Windows. | Загружаются абсолютно все драйверы устройств. |
| **Службы** | Активны только жизненно важные. | Активны все службы Microsoft, отключены только сторонние. |
| **Автозагрузка** | Блокируется на уровне системы. | Отключается пользователем вручную. |
| **Доступ к сети** | Отсутствует (если не выбран режим с сетевыми драйверами). | Доступна в полном объеме. |
| **Интерфейс** | Ограниченный (низкое разрешение, черный фон). | Стандартный интерфейс с привычным разрешением. |

## Вывод
В ходе лабораторной работы были освоены навыки работы с диагностическими инструментами Windows 7. Я изучил методы активации безопасного режима через конфигуратор `msconfig`, а также механизм создания и применения контрольных точек восстановления. Анализ показал, что возврат к более старым точкам затрагивает большее количество установленного ПО. Практический откат системы в безопасном режиме был выполнен успешно, после чего работоспособность ПК в стандартном режиме была полностью восстановлена.

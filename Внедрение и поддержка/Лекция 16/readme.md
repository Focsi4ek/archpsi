# Теоретическая справка: Диагностический (безопасный) режим в Windows

## Понятие безопасного режима

**Безопасный режим (Safe Mode)** представляет собой специализированный вариант старта операционной системы. Его ключевая особенность заключается в том, что система загружается исключительно с базовым набором критически важных компонентов.

Этот режим часто называют **диагностическим**. К нему прибегают в тех случаях, когда Windows работает нестабильно, зависает или уходит в цикличную перезагрузку, и администратору необходимо отделить системные ошибки от конфликтов, вызванных сторонним программным обеспечением или драйверами.

---

## Ключевые особенности режима

*   Инициализируются только те **базовые драйверы и системные службы**, без которых функционирование ОС физически невозможно.
*   **Любые второстепенные процессы**, пользовательские программы и элементы автозагрузки принудительно игнорируются.
*   Многократно **возрастает шанс успешного старта** даже сильно поврежденной системы.
*   Создается изолированная среда, **упрощающая поиск источника сбоев** (будь то кривое обновление, вирус или драйвер).

---

## Практическое применение (возможности)

| Решаемая задача | Механизм действия в Safe Mode |
| :--- | :--- |
| **Удаление вредоносного ПО** | Большинство вирусов прописываются в автозагрузку. В безопасном режиме они не могут активироваться и "спрятаться" в памяти, что позволяет антивирусным сканерам легко их удалить. |
| **Откат системы** | Запуск штатного «Средства восстановления» происходит в стабильной среде, без риска прерывания процесса из-за сторонних конфликтов. |
| **Манипуляции с драйверами** | Поскольку система использует минимальные универсальные драйверы, можно без возникновения критических ошибок (BSOD) удалить или откатить сбойный драйвер видеокарты или другого оборудования. |
| **Аппаратная диагностика** | Работает метод исключения: если в Safe Mode проблема исчезла — виноват софт/драйверы. Если компьютер продолжает зависать или выключаться — проблема кроется в аппаратной части ("железе"). |

---

## Варианты загрузки безопасного режима

| Тип загрузки | Характеристика |
| :--- | :--- |
| **Классический безопасный режим** | Стандартный "голый" вариант. Загружается строгий минимум компонентов. Сетевые протоколы и драйверы адаптеров **отключены** — доступа в интернет и локальную сеть нет. |
| **Safe Mode с поддержкой сети** | К стандартному набору добавляются сетевые службы. Этот вариант идеален, если для ремонта требуется **выход в интернет** (например, чтобы скачать свежий антивирусный сканер или нужный драйвер). |
| **Safe Mode с командной строкой** | Предназначен для продвинутых пользователей. После загрузки системы вместо привычного рабочего стола и Проводника открывается окно **командной консоли (cmd.exe)**. |

# Отчет по лабораторной работе №19
**Тема:** Анализ и управление журналами событий в ОС Windows
**Цель работы:** Изучить механизмы регистрации системных событий и получить практические навыки работы с утилитой «Просмотр событий».

## Теоретическая справка

**Журнал событий Windows (Event Log)** — это специализированная системная база данных, куда операционная система автоматически вносит записи обо всех значимых процессах. Сюда попадают уведомления об успешных операциях (например, старт службы), системные предупреждения и критические ошибки оборудования или ПО.

**Главная задача инструмента:** предоставление администратору структурированной информации для поиска причин программных или аппаратных сбоев.

**Расположение файлов:** Физически логи хранятся на системном диске в каталоге:
`%SystemRoot%\System32\winevt\Logs`
Данные записываются в файлах с расширением `.evtx`. Это **бинарный формат**, поэтому прочитать их через обычный текстовый редактор (например, Блокнот) невозможно.

**Инструмент для чтения:** Утилита «Просмотр событий» (`eventvwr.msc`).
*Запуск:* `Win+R` → `eventvwr.msc` (или через Панель управления → Администрирование).

### Интерфейс утилиты
Окно просмотра событий разделено на несколько функциональных зон:

| Элемент интерфейса | Назначение |
| :--- | :--- |
| **Левая панель** | Дерево каталогов с доступными журналами |
| **Центральная область** | Хронологический список зафиксированных событий |
| **Правая панель (Действия)** | Инструменты для фильтрации, сохранения и привязки задач |
| **Нижний блок** | Детальное описание выбранного в списке события |

### Основные разделы журналов
Система делит события на несколько глобальных узлов:
*   Настраиваемые представления
*   **Журналы Windows** *(главная ветка для системной аналитики)*
*   Журналы приложений и служб
*   Подписки

### Ветка «Журналы Windows» и ее категории
| Имя категории | Описание заносимых данных |
| :--- | :--- |
| **Система** | Сбои и уведомления от самой Windows, ее модулей и драйверов. |
| **Приложения** | Информационные сообщения и ошибки, генерируемые сторонним софтом. |
| **Безопасность** | Аудит доступов: авторизация пользователей, изменение прав, запуск/остановка системных процессов. |
| **Установка** *(доп.)* | Логи процессов инсталляции и обновления компонентов ОС. |
| **Перенаправленные события** *(доп.)* | Удаленно собранные логи с других ПК в локальной сети. |

## Практическая часть

### Шаг 1. Открытие оснастки
1. Вызвал окно выполнения команд (`Win+R`), ввел `eventvwr.msc` и подтвердил запуск.
2. После открытия программы ознакомился с пользовательским интерфейсом: изучил дерево категорий слева и панель доступных действий справа.

### Шаг 2. Настройка фильтра для лога «Система»
1. В древовидном меню слева перешел по пути: **Журналы Windows → Система**.
2. В блоке действий справа кликнул на **«Фильтр текущего журнала»**.
3. Задал следующие условия поиска:
   *   Временной интервал: **Последние 7 дней**
   *   Типы событий (галочки): **Критические**, **Ошибки**, **Предупреждения**
4. Применил настройки кнопкой **ОК**.
*Результат:* В центральном окне отобразилась выборка примерно из 47 событий, соответствующих заданным критериям.

### Шаг 3. Сохранение пользовательского фильтра
1. В правой колонке выбрал пункт **«Сохранить фильтр в настраиваемом представлении»**.
2. В окне сохранения указал имя: **«Система — Ошибки»** и нажал **ОК**.
3. Проверил левую панель: созданный фильтр успешно появился в папке «Настраиваемые представления».

### Шаг 4. Самостоятельная фильтрация лога «Безопасность»
1. Развернул ветку: **Журналы Windows → Безопасность**.
2. Вызвал **«Фильтр текущего журнала»** и установил параметры:
   *   Временной интервал: **Последние 30 дней**
   *   Типы событий: **Предупреждения**, **Сведения**
3. Нажал **ОК** для применения.
4. Сохранил полученную выборку как настраиваемое представление под именем: **«2ИСП9-1 Безопасность»**.

### Шаг 5. Автоматизация (Привязка задачи)
1. В отфильтрованном списке «Безопасность» выделил конкретную запись (согласно выданному варианту).
2. На панели действий справа нажал **«Привязать задачу к событию»**.
3. В открывшемся мастере планировщика:
   *   *(Для ОС Windows 7)* Указал действие **«Отобразить сообщение»**.
   *   Отметил пункт **«Открыть окно Свойства при нажатии кнопки Готово»**.
   *   Пройдясь по вкладкам **«Общие»** и **«Параметры»**, сконфигурировал дополнительные условия выполнения задачи.
4. Завершил настройку кнопкой **Готово**.

### Шаг 6. Работа с журналом «Установка»
1. В навигационном дереве выбрал: **Журналы Windows → Установка**.
2. Вызвал контекстное меню (Правая кнопка мыши) и нажал **«Очистить журнал»**.
3. Содержимое категории было успешно удалено.

### Шаг 7. Конфигурация службы Event Log
1. Открыл менеджер служб: `Win+R` → `services.msc`.
2. Нашел в перечне службу **«Журнал событий Windows»** и открыл ее свойства двойным кликом.
3. Внес изменения:
   *   На вкладке **«Общие»** изменил тип запуска на **«Вручную»**.
   *   На вкладке **«Восстановление»** задал реакцию системы на сбои: первый и второй отказ — **«Перезапуск службы»**; третий и последующие отказы — **«Перезагрузка компьютера»**.
4. Сохранил новую конфигурацию (**Применить** → **ОК**).

## Ответы на контрольные вопросы

### 1. Понятие журналов событий
**Журнал событий** — это встроенный механизм Windows, предназначенный для постоянного мониторинга и записи всех критичных системных процессов. Доступ к этой базе осуществляется через оснастку `eventvwr.msc`. С ее помощью системные администраторы локализуют причины ошибок, отслеживают несанкционированные входы в систему и контролируют успешность установки апдейтов и драйверов.

### 2. Размещение файлов журналов
Файлы логов генерируются в проприетарном бинарном формате `.evtx`, который читается только специализированными системными утилитами.
Стандартный путь хранения на жестком диске:
`%SystemRoot%\System32\winevt\Logs` *(по умолчанию это `C:\Windows\System32\winevt\Logs`)*.

### 3. Структура категории «Журналы Windows»
**Базовые разделы:**
| Название | Назначение |
| :--- | :--- |
| **Система** | Хранит логи операционной системы (падения драйверов, ошибки загрузки модулей ядра). |
| **Приложения** | Собирает отчеты от инсталлированного пользовательского софта (вылеты программ, ошибки доступа). |
| **Безопасность** | Фиксирует события аудита (входы по паролю, изменения политик, манипуляции с файлами). |

**Дополнительные разделы:**
| Название | Назначение |
| :--- | :--- |
| **Установка** | Протоколирует процесс установки новых патчей и компонентов самой Windows. |
| **Перенаправленные события** | Содержит логи, агрегированные с других ПК внутри рабочей группы или домена. |

# Теоретическая справка: Системные журналы событий Windows

## Назначение журнала событий

**Журнал событий (Event Log)** — это централизованный системный инструмент Windows, предназначенный для сбора, хронологического хранения и анализа уведомлений от операционной системы и установленного ПО. В эту единую базу данных записываются все информационные сообщения, системные предупреждения и программные ошибки.

---

## Базовые категории журналов

| Название журнала | Описание содержимого |
| :--- | :--- |
| **Журнал приложений** | Хранит отчеты и логи от установленного пользовательского и стороннего ПО. Какие именно события сюда попадут — решают разработчики каждой конкретной программы. |
| **Журнал безопасности** | Фиксирует события аудита: удачные и неудачные попытки авторизации, а также факты создания, изменения или удаления защищенных файлов. Правила для записи в этот журнал задает системный администратор. |
| **Системный журнал** | Регистрирует события, сгенерированные самой операционной системой: сбои при инициализации драйверов, ошибки служб, проблемы с аппаратными компонентами. |

---

## Классификация регистрируемых событий

Каждая запись в журнале маркируется специальным значком для быстрой визуальной идентификации:

| Тип события | Пиктограмма | Что означает |
| :--- | :--- | :--- |
| **Ошибка** | 🔴 (Крестик в красном кружке) | Фиксирует критические проблемы, сопровождающиеся потерей функционала или данных (например, отказ жизненно важной службы). |
| **Предупреждение** | 🟡 (Восклицательный знак в желтом треугольнике) | Не является критическим сбоем прямо сейчас, но сигнализирует о потенциальных проблемах в будущем (например, исчерпание места на жестком диске). |
| **Сведения (Инфо)** | 🔵 (Буква «i» в белом/синем кружке) | Информационное сообщение о том, что штатная операция завершилась успешно (служба запущена, драйвер подгружен). |
| **Аудит успехов** | ✅ (Ключ / Галочка) | Подтверждает успешное выполнение операции, связанной с безопасностью (например, юзер успешно вошел в свой профиль). |
| **Аудит отказов** | ❌ (Замок / Крестик) | Фиксирует заблокированное действие или неудачную попытку (например, попытка открыть папку, на которую нет прав, или ввод неверного пароля). |

---

## Ключевые особенности и ограничения

*   Служба регистрации событий интегрирована в ядро и **стартует автоматически** вместе с загрузкой ОС.
*   Читать логи **«Системы»** и **«Приложений»** могут любые пользователи, авторизованные на ПК.
*   Доступ к логу **«Безопасности»** имеет исключительно учетная запись **Администратора**.
*   По умолчанию ОС не ведет запись в журнал безопасности (в целях экономии ресурсов). Включить аудит нужных событий можно только через **Групповую политику**.

# Теоретическая справка: Основы операционной системы Linux

## Что такое Linux и история создания

**Автор ядра:** Линус Торвальдс (во время создания — студент Хельсинкского университета).

**Linux** — это семейство свободных Unix-подобных операционных систем, которые строятся на базе одноименного ядра и комплекта системных утилит/библиотек.

Главная философия Linux заключается в открытости и безопасности. Само ядро ОС, а также подавляющее большинство пользовательских программ имеют открытый исходный код и распространяются **совершенно бесплатно** (Open Source).

---

## Понятие дистрибутива

**Дистрибутив** — это уже полностью готовая к использованию операционная система, которая включает в себя ядро Linux, графическую оболочку, программу-установщик и базовый пакет программного обеспечения.

*   Хотя у каждого дистрибутива **свой уникальный установщик**, базовая логика инсталляции и работы системы остается общей.
*   Главные отличия между дистрибутивами заключаются во внешнем виде (графической среде) и наборе софта, установленного «из коробки».
*   Современные дистрибутивы дружелюбны к новичкам: базовую систему можно установить и использовать, не вдаваясь в глубокие технические детали.

Самым известным и массовым дистрибутивом является **Ubuntu**. Его главный девиз: *«Linux для людей»*.

## Архитектура файловой системы

| Термин | Значение в среде Linux |
| :--- | :--- |
| **Корневой каталог (`/`)** | Базовая директория всей ОС. В Linux нет привычных дисков C: или D: — абсолютно все устройства и папки являются ветвями единого дерева, растущего из «корня». |
| **Монтирование** | Процесс программного подключения физического носителя (флешки, раздела диска) к определенной папке в файловой системе ОС. |
| **Точка монтирования** | Обычная пустая папка в корневом дереве, которая после монтирования начинает отображать содержимое подключенного диска. |
| **Размонтирование** | Процедура программного отключения носителя. После этого папка (точка монтирования) снова становится пустой. |
| **Автомонтирование** | Системный механизм, который сам подключает все найденные диски при старте компьютера и безопасно их отключает при завершении работы. |

**Стандартная файловая система:** Часто используется журналируемая файловая система `ext3` (размер кластера варьируется от 1 до 8 КБ).

## Сравнительный анализ: Linux против Windows

| Критерий оценки | Операционная система Windows | Операционная система Linux |
| :--- | :--- | :--- |
| **Безопасность** | Является главной мишенью для 99% вредоносного ПО и вирусов. | Вирусов практически не существует; архитектура прав доступа не позволяет им заразить систему целиком. |
| **Стабильность работы** | Склонна к накоплению ошибок, «синим экранам» и деградации производительности со временем. | Отличается высочайшей надежностью (может работать годами без перезагрузок и переустановки). |
| **Ценовая политика** | Требует покупки дорогостоящей лицензии. | Абсолютно бесплатна для любых целей. |
| **Требования к «железу»** | Постоянно растут с выходом каждого крупного обновления. | Очень гибкие; существуют легковесные версии, отлично работающие на старых ПК. |
| **Обновления** | Зависят исключительно от серверов корпорации Microsoft. | Выпускаются регулярно сообществом разработчиков; исходный код всегда доступен для аудита. |
| **Исходный код** | Проприетарный (закрытый от сторонних глаз). | Открытый (каждый может посмотреть, как работает система). |
| **Лицензионные рамки** | Жесткие ограничения: один ключ на один ПК, запрет на несанкционированное копирование и модификацию. | Свобода действий: можно легально копировать, раздавать друзьям, изменять код и выпускать свои собственные версии. |

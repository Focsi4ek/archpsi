# Конспект: Основы UNIX-подобных операционных систем

## 1. Структура каталогов

В UNIX-подобных системах все каталоги образуют единое дерево с одним корнем. Основные каталоги:

| Каталог | Назначение |
|---|---|
| `/` | Корневой каталог — вершина всей файловой системы |
| `/home` | Домашние каталоги пользователей |
| `/media` | Точки монтирования съёмных носителей (флешки, диски) |
| `/scr` | Скратч-файлы (временные рабочие файлы) |
| `/scr0` | Ещё один каталог для скратч-файлов |

> Скратч-файлы (scratch) — временные файлы, которые программы создают в процессе вычислений и удаляют после завершения работы.

---

## 2. Работа с файлами и каталогами

Основные команды для управления файлами и каталогами:

| Команда | Действие |
|---|---|
| `cd` | Изменяет текущий каталог (change directory) |
| `ls` | Выводит содержимое каталога (list) |
| `cp` | Копирует файлы и каталоги (copy) |
| `mkdir` | Создаёт каталоги (make directory) |
| `mv` | Перемещает файлы и каталоги (move) |
| `rm` | Удаляет файлы (remove) |
| `rmdir` | Удаляет каталоги (remove directory) |

---

## 3. Чувствительность к регистру

**Linux чувствителен к регистру символов!**

Это значит, что заглавные и строчные буквы в именах считаются разными. Например, файлы с именами `H2O.gjf` и `h2o.gjf` — это **два разных файла**, которые могут спокойно лежать в одном каталоге.

Это важно учитывать при наборе команд и имён файлов: ошибка в регистре приведёт к тому, что система не найдёт нужный файл.

---

## 4. Командная строка

Полезные клавиши при работе в терминале:

- **Стрелки ↑ и ↓** — перемещение по истории ранее введённых команд
- **Tab** — автодополнение: дописывает начатую команду или имя файла
- **Tab дважды** — выводит все возможные варианты дополнения
- **Ctrl + C** — прерывает (завершает) выполнение текущей команды

---

## 5. Текстовый редактор nano

`nano` — простой консольный текстовый редактор.

- Текст, **выделенный мышью**, автоматически попадает в буфер обмена
- Для вставки из буфера используется сочетание **Shift + Ins**

---

## 6. Файловый менеджер Midnight Commander

Midnight Commander (MC) — двухпанельный файловый менеджер с текстовым интерфейсом.

- **Запуск:** команда `mc`
- **Ctrl + O** — временно скрыть панели и вернуться к командной строке (и обратно)
- **F10** — выход из программы
- Основные операции очень похожи на файловый менеджер **Far** (привычный по Windows)

---

## Практическое задание

1. Создать пользователя:
   - Логин: `notuser`
   - Пароль: `123`
2. Создать в домашнем каталоге каталог `H2o` и текстовый файл `h2O.txt`
3. Скопировать файл `h2O.txt` в каталог `H2o`
4. Посмотреть содержимое каталога `H2o`

**Примерная последовательность команд для выполнения** (пункты 2–4, в домашнем каталоге):

```bash
mkdir H2o
touch h2O.txt
cp h2O.txt H2o/
ls H2o
```

> Обрати внимание на регистр: `H2o` (каталог) и `h2O.txt` (файл) — имена с разным регистром, и Linux воспринимает их именно так, как написано. Это задание как раз проверяет понимание чувствительности к регистру из раздела 3.

---

## Краткое резюме

- Файловая система UNIX — единое дерево с корнем `/`; ключевые каталоги: `/home`, `/media`, `/scr`
- Базовый набор команд для файлов и каталогов: `cd`, `ls`, `cp`, `mv`, `mkdir`, `rmdir`, `rm`
- Linux **чувствителен к регистру** — `File` и `file` это разные имена
- Командная строка ускоряется историей (стрелки), автодополнением (Tab) и прерыванием (Ctrl+C)
- Для редактирования текста есть `nano`, для управления файлами — Midnight Commander (`mc`)

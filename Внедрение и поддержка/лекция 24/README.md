## 1. Создание и объединение файлов

Создал два текстовых файла с помощью `echo` и перенаправления вывода, затем объединил их в один файл командой `cat`.

```bash
echo "Первый файл" > file1.txt
echo "Второй файл" > file2.txt
cat file1.txt file2.txt > file_combined.txt
cat file_combined.txt
```

**Вывод:**
```
Первый файл
Второй файл
```

![Создание и объединение файлов](screenshots/01-create-merge.png)

**Пояснение:**
- `>` создаёт файл (или перезаписывает его) и помещает туда вывод команды
- `cat file1.txt file2.txt > file_combined.txt` последовательно выводит содержимое обоих файлов и перенаправляет результат в новый файл — так происходит конкатенация (объединение)
- `cat file_combined.txt` без перенаправления просто печатает содержимое файла на экран — видно, что оба файла склеились в правильном порядке

- `ls -l` выводит полную информацию: права доступа, число ссылок, владельца, группу, размер в байтах, дату модификации и имя файла

## 3. Изменение прав доступа

Добавил право на выполнение для группы и остальных пользователей.

```bash
cd lab224_dir
chmod g+x,o+x file1.txt file2.txt file_combined.txt
ls -l
```

**Вывод (после изменения):**
```
total 12
-rw-rwxr-x 1 vboxuser vboxuser 22 Jun 21 20:18 file1.txt
-rw-rwxr-x 1 vboxuser vboxuser 22 Jun 21 20:18 file2.txt
-rw-rwxr-x 1 vboxuser vboxuser 44 Jun 21 20:19 file_combined.txt
```

![Изменение прав доступа](screenshots/03-chmod.png)

**Пояснение:**
- `chmod g+x,o+x` — символьный режим записи прав: `g+x` добавляет право выполнения группе, `o+x` добавляет его остальным пользователям; запятая позволяет задать несколько изменений в одной команде
- Права владельца (`rw-`) не изменились — задание требовало добавить `x` только группе и остальным
- Итоговые права `-rw-rwxr-x`: владелец — чтение/запись, группа — чтение/запись/выполнение, остальные — чтение/выполнение

> Примечание: на скриншоте видна строка `^[[200~ls -l~` с ошибкой `command not found` — это технический артефакт терминала при вставке текста (bracketed paste mode), а не ошибка выполнения лабораторной работы. Команда `ls -l` была сразу после этого введена и выполнена корректно.

## Контрольные вопросы

### 1. Что считается файлами в ОС Linux?

В Linux действует принцип «всё есть файл» (*everything is a file*). Файлами считаются не только привычные текстовые или бинарные данные на диске, но и:

- **обычные файлы** — тексты, программы, архивы и т.д.;
- **каталоги (директории)** — особый тип файла, который содержит список имён файлов и ссылок на них;
- **специальные файлы устройств** — каждое периферийное устройство (диск, принтер, терминал) представлено файлом в каталоге `/dev`. Это позволяет программам обращаться к устройствам так же, как к обычным файлам — через стандартные операции чтения/записи, без знания специфики конкретного «железа»;
- **символические связи (symlinks)** — файлы, которые хранят не данные, а путь к другому файлу;
- **именованные каналы, сокеты** — используются для межпроцессного взаимодействия.

Такой единый подход к разным сущностям через файловый интерфейс упрощает написание программ: код, который читает данные из файла, без изменений может читать данные с устройства.

### 2. Объясните назначение связей с файлами и способы их создания.

**Назначение связей (links)** — дать одному физическому файлу (точнее, его данным на диске) несколько имён, возможно в разных каталогах, без дублирования самих данных. Это экономит место на диске и позволяет обращаться к одному и тому же файлу из разных мест файловой системы под разными именами.

Существует два вида связей:

- **Жёсткая связь (hard link)** — создаётся командой:
  ```bash
  ln <имя_файла_1> <имя_файла_2>
  ```
  Жёсткая связь — это второе имя того же самого файла (той же inode, то есть той же области данных на диске). Файл физически не удаляется, пока существует хотя бы одна жёсткая связь на него. Жёсткие связи не могут пересекать границы разных файловых систем (разделов/дисков) и не могут указывать на каталоги.

- **Символическая связь (symbolic link, symlink)** — создаётся командой с флагом `-s`:
  ```bash
  ln -s <имя_файла_1> <имя_файла_2>
  ```
  Символическая связь — это отдельный, особый файл, который хранит в себе путь к файлу-цели. При обращении к симлинку система автоматически «разворачивает» его и работает с файлом, на который он указывает (например, при выводе содержимого симлинка через `cat` отображаются данные файла-цели, а не текст самого симлинка). Символические связи могут указывать на файлы в других файловых системах и на каталоги, но если файл-цель удалён, симлинк становится «битым» (broken link) и перестаёт работать.

### 3. Что определяет атрибуты файлов и каким образом их можно просмотреть и изменить?

**Атрибуты файла** определяют его тип и права доступа к нему для трёх категорий пользователей: владельца, группы-владельца и всех остальных. Для каждой категории задаётся набор из трёх разрешений:

- `r` — чтение;
- `w` — запись;
- `x` — выполнение (для файла) или вход в каталог (для директории);
- `-` — отсутствие соответствующего права.

Первый символ в строке атрибутов указывает тип файла: `-` — обычный файл, `d` — каталог, `l` — символическая связь, `b` — блочное устройство, `c` — символьное устройство.

**Просмотр атрибутов** — командой:
```bash
ls -l [имя_файла_или_каталога]
```
Вывод показывает строку из 10 символов вида `-rwxr-xr--`, а также число связей, владельца, группу, размер, дату модификации и имя файла.

**Изменение атрибутов** — командой `chmod`:
```bash
chmod <коды_доступа> <имя_файла>
```
Коды доступа задаются в двух формах:

- **символьная форма**: указывается категория (`u` — владелец, `g` — группа, `o` — остальные, `a` — все) и операция (`+` добавить, `-` убрать, `=` установить точно) с правом (`r`, `w`, `x`). Например: `chmod g+rw,o+r file.txt`;
- **числовая (восьмеричная) форма**: каждое право кодируется битом (r=4, w=2, x=1), сумма битов даёт цифру для каждой из трёх категорий. Например, `chmod 754 file.txt` означает `rwxr-xr--`.

После изменения рекомендуется проверять результат командой `ls -l`.

### 4. Какие методы создания и удаления файлов, каталогов вы знаете?

**Создание файлов:**
- `> filename` — создаёт пустой файл (или перезаписывает существующий), также используется для перенаправления вывода в файл;
- `cat file1 file2 > file3` — создаёт новый файл как результат объединения (конкатенации) других файлов;
- `touch filename` — создаёт пустой файл, если он не существует (или обновляет время последнего доступа, если существует);
- любой текстовый редактор (`nano`, `vi`, `vim`) также создаёт файл при сохранении.

**Удаление файлов:**
```bash
rm file1 file2 file3
```
удаляет один или несколько указанных файлов.

**Создание каталогов:**
```bash
mkdir namedir
```
создаёт новый каталог с указанным именем (с флагом `-p` можно создать сразу всю цепочку вложенных каталогов).

**Удаление каталогов:**
```bash
rmdir dir_1 dir_2
```
удаляет каталоги, но только если они пустые;
```bash
rm -r dir_1 dir_2
```
удаляет каталоги рекурсивно вместе со всем содержимым (используется, если каталог не пуст).

### 5. В чём заключается поиск по шаблону?

Поиск по шаблону выполняется командой `grep` и заключается в построчном просмотре содержимого файла (или нескольких файлов) с целью найти строки, соответствующие заданному шаблону (последовательности символов или регулярному выражению):

```bash
grep [ключи] <шаблон_поиска> <имя_файла>
```

По умолчанию `grep` печатает все строки, в которых встретился шаблон. Поведение можно настраивать ключами, например:

- `-v` — инвертировать поиск: вывести строки, **не** содержащие шаблон;
- `-i` — игнорировать регистр символов при сравнении;
- `-l` — вывести только имена файлов, где найден шаблон (без самих строк);
- `-c` — вывести только количество совпавших строк;
- `-n` — пронумеровать строки в выводе.

Пример: `grep -l student *.txt` найдёт среди всех `.txt`-файлов текущего каталога те, что содержат слово «student», и выведет их имена.

Поиск по шаблону применяется не только к содержимому файлов, но и, например, для фильтрации вывода других команд — как в примере `who | grep tty23`, где `grep` ищет в выводе команды `who` строку, относящуюся к конкретному терминалу.

### 6. Какой командой можно получить список работающих пользователей и сохранить его в файле?

Список пользователей, работающих в системе в данный момент, выводит команда:
```bash
who
```
Она показывает имя пользователя, терминал, с которого выполнен вход, и время входа в систему.

Чтобы сохранить этот список в файл, используется перенаправление вывода:
```bash
who > users.txt
```

Если нужно не создать новый файл, а дополнить уже существующий — используется двойной знак `>>`:
```bash
who >> users.txt
```

Также для получения списка пользователей можно использовать команды `w` (показывает дополнительно, чем заняты пользователи) или `users`, результат которых аналогично перенаправляется в файл через `>`.

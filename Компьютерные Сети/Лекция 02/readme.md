## ⏳ 1. История развития сетевых технологий

Эволюция вычислительных сетей представляет собой путь от разрозненных локальных экспериментов до создания единой глобальной информационной среды.

**Ключевые исторические вехи:**

| Период | Значимое событие |
| :--- | :--- |
| **Начало 1970‑х** | Создание первых нестандартных ЛВС, ставших фундаментом для последующего объединения ЭВМ. |
| **1971 год** | Рэй Томлинсон (инженер BNN) разрабатывает первую в мире программу для обмена электронной почтой. |
| **1973 год** | Роберт Меткалф (Xerox) формулирует концепцию и предлагает название для технологии **Ethernet**. |
| **1974 год** | Винтон Серф и Роберт Кан публикуют спецификацию протокола управления передачей — **TCP**. |
| **1978–1979 гг.** | Создание универсальной эталонной модели **OSI** для стандартизации сетевого взаимодействия. |
| **Начало 1980‑х** | Разработка и внедрение стека **TCP/IP**, сделавшего узлы независимыми от физической среды. Массовый переход ARPANET на TCP/IP, зарождение современного Интернета. |
| **Середина 1980‑х** | Стандартизация базовых технологий ЛВС: **Ethernet** (1980), **Token Ring** (1985) и оптической **FDDI** (1985). |
| **2 ноября 1988 г.** | Первая глобальная кибератака («червь Морриса»), парализовавшая около 6 000 серверов ARPANET. Итогом стало создание координационного центра **CERT** для борьбы с сетевыми угрозами. |
| **Рубеж 1980–1990‑х** | Планомерная замена устаревшей ARPANET на скоростную сеть Национального научного фонда США (**NSFNet**). |
| **1991–1992 гг.** | Тим Бернерс-Ли (CERN) создает технологию **World Wide Web (WWW)**. |
| **1994 год** | Утверждение протокола соединения «точка-точка» (**PPP**). |
| **1995 год** | Трансформация NSFNet в коммерческую опорную сеть. Появление независимых интернет-провайдеров (**ISP**) и формирование глобальной сети Интернет в ее текущем статусе. |

## 🏗️ 2. Теоретические основы: Модели и протоколы

Для того чтобы оборудование от различных вендоров могло корректно обмениваться данными, используются стандартизированные эталонные модели.

**Базовые понятия:**
* **Сетевой протокол:** строго регламентированный свод правил и форматов, определяющий порядок взаимодействия устройств в сети.
* **Стек протоколов:** многоуровневая (иерархическая) структура протоколов, работающих совместно для обеспечения связи.

### 2.1. Эталонная модель OSI

Разработана Международной организацией по стандартизации (ISO) в начале 1980-х годов как универсальный стандарт для архитектуры вычислительных сетей с коммутацией пакетов.

**7 уровней модели OSI (снизу вверх):**

1. 🔌 **Физический (Physical):**
   * **Функция:** передача неструктурированных битовых потоков через физическую среду (кабели, радиоволны).
   * **Специфика:** реализуется исключительно на аппаратном уровне (например, стандарт 100Base-T4).

2. 🔗 **Канальный (Data Link):**
   * **Функция:** доставка данных между узлами в пределах одного сегмента локальной сети или точка-точка в WAN.
   * **Блок данных (PDU):** кадр (frame).
   * **Особенности:** именно здесь зарождается коммутация. Отвечает за проверку целостности кадров и физическую адресацию (MAC). Работает на уровне коммутаторов и сетевых карт.

3. 🌐 **Сетевой (Network):**
   * **Функция:** объединение различных сетей и логическая адресация.
   * **Блок данных (PDU):** пакет (packet).
   * **Главная задача:** маршрутизация трафика (поиск оптимального пути). Протоколы: IP, IPX, OSPF, BGP. Работает на уровне маршрутизаторов.

4. 🚚 **Транспортный (Transport):**
   * **Функция:** сквозная передача данных между конечными хостами с заданным уровнем надежности.
   * **Блок данных (PDU):** сегмент (TCP) или дейтаграмма (UDP).
   * **Задачи:** управление потоком, гарантированная доставка, мультиплексирование портов.

5. 🤝 **Сеансовый (Session):**
   * **Функция:** управление логическими сессиями между приложениями (установка, поддержание, синхронизация и разрыв связи). Пример: SSH.

6. 🎭 **Представительный (Presentation):**
   * **Функция:** преобразование форматов данных для прикладных программ.
   * **Задачи:** кодирование, компрессия (сжатие) и криптографическая защита (SSL/TLS).

7. 💻 **Прикладной (Application):**
   * **Функция:** предоставление пользовательским приложениям прямого доступа к сетевым службам.
   * **Блок данных (PDU):** сообщение (message).
   * **Протоколы:** HTTP, FTP, SMB, SMTP.

### 2.2. Стек TCP/IP (Модель DoD)

Практичная и более компактная модель из четырех уровней, созданная Министерством обороны США. Является основой современного Интернета.

**Соотношение с моделью OSI:**
1. **Прикладной уровень (Application):** поглощает прикладной, представительный и сеансовый уровни OSI.
2. **Транспортный уровень (Transport):** полностью аналогичен 4-му уровню OSI (TCP/UDP).
3. **Сетевой уровень (Internet):** соответствует сетевому уровню OSI (маршрутизация, IP).
4. **Уровень сетевого доступа (Network Access):** объединяет функции канального и физического уровней OSI.

## 💻 3. Архитектура прикладного уровня

Прикладные протоколы задают правила общения конечных программ.

**Основные архитектурные паттерны:**
* **«Клиент-сервер»:** один участник (клиент) инициирует запрос, другой (сервер) его обрабатывает и возвращает результат (веб-браузеры, почта).
* **Одноранговая (P2P):** децентрализованная архитектура, где все узлы равноправны и выступают одновременно и клиентами, и серверами (торренты).

**Сетевые сокеты:**
Точка подключения в сети определяется **сокетом** — программным шлюзом между приложением и транспортным протоколом.
Сокет состоит из:
1. **IP-адреса** (или доменного имени хоста).
2. **Номера порта** (идентификатор службы, например, 80 для веба, 25 для почты).

**Что стандартизирует любой прикладной протокол:**
* Типы допустимых сообщений (запросы/ответы).
* Формат и синтаксис сообщений.
* Семантику (значение) каждого поля.
* Алгоритмы поведения приложений при получении тех или иных сообщений.

## 🔍 4. Система доменных имён (DNS)

Служба DNS предназначена для трансляции читаемых доменных имен в IP-адреса машин.

**Развитие систем именования:**
* **Плоские (линейные) имена:** использовались на заре сетей, без вложенности (например, `SERVER1`).
* **Иерархические имена (домены):** древовидная структура, применяемая сегодня (`mail.google.com`).

**Домены верхнего уровня (TLD):**
* **Общего назначения (gTLD):** .com, .net, .org, .info.
* **Национальные (ccTLD):** .ru, .kz, .us, .uk.
* **Спонсируемые:** выделенные для конкретных организаций или отраслей.

**Принципы работы DNS:**
* Архитектура представляет собой распределенную базу данных по независимым зонам ответственности.
* За поиск адресов отвечает компонент **DNS-резолвер**.
* **Основные типы записей:** **A** (соответствие имени и IPv4-адреса), **NS** (указатель на авторитетный сервер), **CNAME** (каноническое имя/псевдоним), **MX** (почтовый шлюз).
* **Обратная зона** (`in-addr.arpa`) позволяет узнать доменное имя, имея на руках только IP-адрес.

**Алгоритм разрешения имени в Windows:**
1. Проверка совпадения с локальным именем ПК.
2. Поиск в локальном кэше DNS (включая записи из файла `hosts`).
3. Запрос к предпочитаемому внешнему DNS-серверу.
4. Если имя плоское (NetBIOS) — поиск в кэше NetBIOS.
5. Запрос к серверу WINS.
6. Рассылка широковещательного запроса (Broadcast).
7. Проверка устаревшего файла `lmhosts`.
8. Возврат ошибки при неудаче на всех этапах.

*Расположение файла hosts:* `%SystemRoot%\System32\drivers\etc\hosts` (в Windows) или `/etc/hosts` (в Linux).

## 📡 5. Обзор ключевых прикладных протоколов

### 📁 5.1. FTP (File Transfer Protocol)
* **Цель:** стандартизированный обмен файлами.
* **Особенности:** поддерживает как анонимный вход, так и авторизацию по логину/паролю.
* **Механика:** использует два параллельных TCP-канала: один для управляющих команд (порт 21), второй — для физической передачи данных.

### 📜 5.2. TFTP (Trivial FTP)
* **Цель:** максимально упрощенная передача файлов без накладных расходов.
* **Особенности:** нет механизмов авторизации и шифрования.
* **Механика:** работает поверх протокола UDP (порт 69). Файлы передаются строгими блоками по 512 байт.
* **Применение:** загрузка ОС по сети (PXE), бекап прошивок на сетевых коммутаторах и маршрутизаторах.

### ✉️ 5.3. Электронная почта
Работает на базе архитектурной модели X.400:
* **MUA (Mail User Agent):** почтовая программа конечного пользователя.
* **MTA (Message Transfer Agent):** сервер-пересыльщик, осуществляющий маршрутизацию письма в сети.
* **MDA (Message Delivery Agent):** агент, который раскладывает входящую почту по ящикам пользователей.
* **MS (Message Store):** дисковое хранилище писем.

### 🌍 5.4. HTTP (HyperText Transfer Protocol)
Основополагающий протокол Интернета. Предназначен для транспортировки гипертекста (HTML-страниц) и мультимедиа. Функционирует по модели «клиент-сервер» (запрос-ответ) поверх протокола TCP. Является фундаментом Всемирной паутины (WWW).
